E2E for Dymension + Kaspa Bridge

Gist
    Hyperlane with synthetic token on hub
    Kaspa does not have smart contracts 
    Run relayer AND validator (not just relayer like in Ethereum/Solana tests)


Requirements
    Require custom agent binaries (`-cargo build --release --bin relayer --bin validator`)
    Require to run local Kaspa full node!! (How, Art?)
    Require testnet tokens (faucet https://faucet-tn10.kaspanet.io/)
    Require REST API client (https://api-tn10.kaspa.org/doc, be careful to avoid becoming rate limited!) 
    Require local hub instance
